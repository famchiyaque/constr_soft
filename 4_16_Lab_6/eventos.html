Preguntas/Respuestas

Al ingresar la contrasena correcta, puedes ver la pagina de venta de 3 cosas

  1. ¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?
  2. Porque puede que el input no este en forma correcto, y al mandarlo al backend, puede causar errores que hacen que caiga el sistema, o si tienes error handling, mandara un status error como 200 por ejemplo. Es mejor validarlo antes en el client-side, donde no hay consecuencia grave por inputs mal ingresados.
  3. ¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?
  4. Primero, puedes no permitir el uso de JavaScript en tu navegador, que desactivara todo el client-side js que trae la pagina. No siempre es con malas intenciones, puede ser por razones de seguridad contra paginas maliciosas. Tambien puedes ir a dev tools y borrar las partes del script que quieras para la duracion de tu sesion.
  5. Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?
  6. Es una buena practica como primer linea de defensa contra los hackers. Para la gran mayoria va a ayudar a evitar errores en el backend, pero el backend tambien debe contar con medidas de seguridad.
