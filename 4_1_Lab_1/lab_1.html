  • Cual es la diferencia entre el internet y la www?

    El internet es la infraesctructura que permite la conextion entre dispositivos en todo el mundo, mientras la world wide web es una parte del internet, donde esa conexcion es a traves de paginas web y ciertos protocolos, como tcp.

  • Cuales son las partes de un URL?

    Hay 3 partes principales, pero pueden haber mas. Primero, el protocolo- como 'http', o 'https'. Luego viene el domain name- 'www.movies.com', o 'amlo.github.io'. Esta es la direccion unica del servidor. Finalmente hay el path, que determina el subdirectorio del servidor, por ejemplo, 'www.drive.google.com/drive. En ese caso, 'drive' es parte del domain name, pero como un sub-domain de google.com.

  • Cual es el proposito de los metedos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

    Un metodo GET pide los headers y el cuerpo, a traves de un API. Un HEAD es lo mismo pero solo pide los headers, para ocupar menos bandwith cuando no quiere el cuerpo. En cambio un POST manda informacion a traves del API. Y un PUT actualiza el data el servidor que lo recibe, pero no puede agregar data como el POST. PATCH tambien es para actualizaciones. Y DELETE es para borrar informacion.

  • ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

    Seria un POST, porque estas mandando informacion al servidor para ser verificado.

  • ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

    Es un GET request, porque estas pidiendo los headers, y tambien el cuerpo de la pagina que trae todo lo que se ve. Un HEAD solo regresa los headers (metadata).

  • Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

    Una respuesta con codigo 200 significa que fue un success. Que todo salio bien pues.

  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

    El codigo 404 y otros de los 400s indican errores por parte del cliente. Si no tienes conexion wifi, cualquier request GET te va a mandar el error 404. Entonces no es responsabilidad del desarrollador.

  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

    El codigo 500 indica un error general del servidor. Entonces si, es problema del desarrollador.

  • ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

    Significa que ese atributo o elemento ha sido reemplazado por un nuevo. Unos ejemplos son: '<acronym>', '<big>', '<dir>', o '<frameset>'.

  • ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

    HMTL5 introdujo muchas cosas para avanzar y facilitar la funcion de html. Trajo nuevas semanticas, nuevos tipos de input, como email o url, la integracion directa de audio y video, y varios APIs incluyendo el de GeoLocation, y el de WebStorage.

  • ¿Qué componentes de estructura y estilo tiene una tabla?

    Una tabla esta compuesta de renglones y columnas. Puedes usar un elemento tabla, y por dentro poner th, que es 'table header' para el primer renglon, y luego puedes poner tr, para cada 'table row'. Automaticamente se les agrega ciertos estilos, como borders de gris, y chance incluso 'text-align: center;' o 'align-items: center;' tambien, no me acuerdo bien.

  • ¿Cuáles son los principales controles de una forma HTML5?

    Hay muchos pero incluyen el tipo input, que acepta texto, contrasenas, emails, numeros de telefono y mas, tambien hay el tipo select para cuando quieres dar al usuario opciones limitadas, el tipo file para que el usuario agregue un archivo, button para que puedan dar clic ahi y que pase algo, muchas veces una funcion 'onClick' que llama una funcion de js.

  • ¿Cuál es el ciclo de vida de los sistemas de información? ¿Cuál es el ciclo de desarrollo de sistemas de información?

    Empieza con un analisis de la complejidad y dificultad que seria crear un nuevo sistema. Luego definen los requisitos, luego lo disenan, luego le hacen testing al diseno, y si sale bien, lo despliegan. Despues de que lo desplieguen, tienen que dedicar personas al mantenimiento del sistema, ademas implementar entrenamiento para los que lo van a estar usando. Mejoras, cambios, y actualizaciones seran necesarios y si hay la posibilidad, lo pueden crecer y hacer grande o agregar nuevas funciones o datos a recopilar.

    El SDLC, o software development life cycle, es lo mismo pero termina al desplegar el sistema.
